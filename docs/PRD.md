## 1. 專案概述

### 1.1 背景與動機

現代上班族每月有固定薪資收入與多樣化的支出項目（房租、水電、餐飲、交通、訂閱服務等），但往往缺乏系統化的記帳習慣，導致月底不知道錢花去哪裡、忘記繳費而產生滯納金等問題。本系統旨在提供一個簡潔、直覺的個人記帳工具，協助使用者輕鬆記錄收入與支出、掌握餘額變化，並透過繳費提醒與常用模板功能降低記帳門檻。

### 1.2 目標用戶

| 屬性 | 描述 |
|------|------|
| **角色** | 上班族（25–45 歲） |
| **特徵** | 有固定月薪收入、每月有多項固定與變動支出 |
| **痛點** | 記帳繁瑣易中斷、忘記繳費期限、不清楚每月收支狀況 |
| **使用情境** | 下班後或午休時用電腦快速記帳、月底查看收支報表 |

### 1.3 核心價值主張

- **簡單快速**：透過常用模板一鍵建立交易紀錄，降低記帳摩擦力
- **不再忘繳費**：繳費提醒功能確保帳單準時繳納，避免滯納金
- **財務透明**：即時餘額統計與圖表，讓使用者隨時掌握財務狀況

## 2. 功能需求

### 2.1 功能總覽

| 編號 | 功能名稱 | 說明 |
|------|----------|------|
| F1 | 繳費提醒 | 設定定期帳單提醒（如房租、電費、信用卡），到期前於首頁顯示提醒 |
| F2 | 記錄收入來源 | 新增收入紀錄，包含金額、來源分類、日期與備註 |
| F3 | 記錄支出 | 新增支出紀錄，包含金額、分類、日期與備註 |
| F4 | 顯示餘額統計 | 首頁儀表板顯示總餘額、本月收入、本月支出及收支趨勢圖表 |
| F5 | 常用模板 | 建立與管理常用交易模板（如「午餐 $100」），一鍵套用快速記帳 |
| F6 | 交易紀錄管理 | 瀏覽、搜尋、編輯、刪除歷史交易紀錄 |
| F7 | 分類管理 | 自訂收入與支出的分類標籤 |

### 2.2 使用者故事

#### F1 — 繳費提醒

- 作為 **上班族**，我希望 **能設定每月固定帳單的繳費日期與金額**，以便 **在到期前收到提醒，避免忘記繳費產生滯納金**。
- 作為 **上班族**，我希望 **在首頁看到即將到期的帳單清單**，以便 **快速掌握近期需要繳納的費用**。
- 作為 **上班族**，我希望 **繳費後能一鍵標記為已繳**，以便 **區分已繳與未繳的帳單**。

#### F2 — 記錄收入來源

- 作為 **上班族**，我希望 **能新增一筆收入紀錄並選擇來源分類（如薪資、獎金、兼職、投資收益）**，以便 **清楚了解各項收入的組成**。
- 作為 **上班族**，我希望 **每筆收入能記錄日期與備註**，以便 **日後回查時能快速辨識該筆收入的來源**。

#### F3 — 記錄支出

- 作為 **上班族**，我希望 **能新增一筆支出紀錄並選擇分類（如餐飲、交通、娛樂、日用品）**，以便 **追蹤各類別的花費狀況**。
- 作為 **上班族**，我希望 **每筆支出能記錄日期與備註**，以便 **日後回查時能快速辨識該筆支出的用途**。

#### F4 — 顯示餘額統計

- 作為 **上班族**，我希望 **在首頁看到目前的總餘額**，以便 **隨時掌握自己的財務狀況**。
- 作為 **上班族**，我希望 **看到本月的收入總額與支出總額**，以便 **了解本月的現金流狀態**。
- 作為 **上班族**，我希望 **看到收支趨勢的圖表（如折線圖或長條圖）**，以便 **觀察每月收支變化的趨勢**。

#### F5 — 常用模板

- 作為 **上班族**，我希望 **能將常見的交易（如每天午餐、通勤費用）儲存為模板**，以便 **下次記帳時一鍵套用，省去重複輸入的麻煩**。
- 作為 **上班族**，我希望 **能管理（編輯、刪除）我的常用模板**，以便 **保持模板清單的整潔與正確**。

#### F6 — 交易紀錄管理

- 作為 **上班族**，我希望 **能瀏覽所有歷史交易紀錄並依日期或分類篩選**，以便 **快速找到特定的交易**。
- 作為 **上班族**，我希望 **能編輯或刪除已建立的交易紀錄**，以便 **修正輸入錯誤的資料**。

#### F7 — 分類管理

- 作為 **上班族**，我希望 **能自訂收入與支出的分類名稱**，以便 **依照個人習慣進行分類記帳**。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Flask（Python） |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML5 + CSS3 + JavaScript |
| 部署方式 | 本機開發環境（localhost） |

### 3.2 效能考量

- 頁面載入時間應在 **2 秒以內**
- 資料庫查詢應在 **500 毫秒以內** 回應
- 單一使用者使用，無需考慮高併發

### 3.3 安全考量

- 所有表單輸入皆需進行 **輸入驗證與清理**，防止 SQL Injection 與 XSS 攻擊
- 金額欄位需驗證為 **正數數值**
- 日期欄位需驗證為 **有效日期格式**

### 3.4 使用者體驗

- 介面需支援 **繁體中文**
- 採用 **響應式設計**，適配桌機與平板螢幕
- 操作流程需簡潔直覺，**3 步內完成一筆記帳**
