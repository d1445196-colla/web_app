# 產品需求文件（PRD）— 即時標記錄音系統

> **文件版本：** v1.0
> **建立日期：** 2026-05-19
> **狀態：** 初稿

---

## 1. 專案概述

### 1.1 背景與動機

在訪談、會議、田野調查等情境中，錄音是最基礎的資料蒐集方式。然而，傳統的錄音工具僅提供「開始 / 暫停 / 停止」等基本操作，使用者在錄製完成後需要反覆回聽整段音訊，才能找到關鍵片段，導致事後整理耗時費力。

本專案旨在打造一套**結合即時標記與語音辨識的錄音系統**，讓使用者在錄音當下就能標記關鍵時間點、標註故事段落與不清晰處，大幅縮短事後資料整理流程，並為後續的語音轉寫、AI 摘要、訪談管理等進階功能提供堅實的資料基礎。

### 1.2 目標用戶

| 用戶類型 | 說明 |
|---------|------|
| 質性研究人員 | 進行訪談、焦點團體等需要錄音並事後分析的研究者 |
| 記者 / 媒體工作者 | 需要快速標記受訪者關鍵回答的採訪人員 |
| 會議記錄者 | 負責整理會議重點與行動項目的行政人員或 PM |
| 學生 / 講座聽眾 | 希望在課堂或演講中標記重點以利複習的學習者 |
| 播客製作者 | 需要標記精彩片段以利後製剪輯的內容創作者 |

### 1.3 核心價值主張

> **「錄音的當下就是整理的開始」**——透過即時標記與視覺化波形，將被動的錄音行為轉化為主動的資料標記過程，讓每一段錄音都帶著結構化的後設資料誕生。

### F1：即時音量波形視覺化

**使用者故事：**
> 作為**錄音使用者**，我希望**在畫面中央看到即時的音量波形圖**，以便**確認麥克風正常收音、錄音正在進行中，並對環境音量有直覺的掌握**。

**詳細說明：**

- 波形圖應佔據畫面中央核心區域，成為主要視覺焦點
- 即時反映麥克風輸入的音量振幅，延遲不超過 100ms
- 提供波形的顏色與動畫效果，讓使用者一眼就能辨別錄音狀態
  - 錄音中：波形動態跳動（例如漸層色彩）
  - 暫停中：波形靜止或以灰色呈現
  - 未開始：顯示靜態基準線
- 波形隨時間軸向左滾動，持續呈現最近數秒的音量變化

---

### F2：錄音計時器

**使用者故事：**
> 作為**錄音使用者**，我希望**在畫面頂部清晰看到錄音已持續的時間**，以便**掌握錄音進度，控制訪談或會議的節奏**。

**詳細說明：**

- 顯示格式為 `HH:MM:SS`（時:分:秒），字型大且醒目
- 位於畫面頂部正中央位置
- 錄音中持續計時，暫停時計時器暫停且有視覺提示（例如閃爍或變色）
- 停止時顯示最終錄音時長

---

### F3：可自訂種類的即時重點標記

**使用者故事：**
> 作為**錄音使用者**，我希望**在錄音過程中隨時點選預設或自訂的標記按鈕，將當前時間點標記為不同種類的重點**，以便**事後能依據標記類型快速定位到特定片段，不必從頭回聽**。

**詳細說明：**

- 系統預設標記種類（使用者可修改）：

  | 標記種類 | 圖示建議 | 用途說明 |
  |---------|---------|---------|
  | 🔑 關鍵重點 | 星號 / 鑰匙 | 標記重要觀點或核心回答 |
  | 📖 故事 | 書本 | 標記受訪者開始講述個人故事的段落 |
  | ❓ 不清晰 | 問號 | 標記語音模糊、需確認的段落 |
  | ⚡ 行動項目 | 閃電 | 標記需要後續追蹤的事項 |
  | 💡 靈感 | 燈泡 | 標記臨時出現的想法或追問方向 |

- 使用者可**新增、編輯、刪除**標記種類（自訂名稱、顏色、圖示）
- 每次標記時自動記錄：
  - 標記時間戳（精確到秒）
  - 標記種類
  - 可選的簡短備註（點選標記後彈出輸入框，可跳過）
- 標記按鈕設計要求：
  - 按鈕尺寸夠大，支援快速點擊（避免影響錄音過程的注意力）
  - 點擊後有明確的視覺回饋（例如漣漪效果 + 計數器 +1）
  - 排列在波形圖下方，方便單手操作
- 標記後，波形圖上對應的時間位置應出現視覺標示（如彩色旗標或圓點）

---

### F4：錄音控制

**使用者故事：**
> 作為**錄音使用者**，我希望**透過清晰的按鈕控制錄音的開始、暫停、繼續與停止**，以便**靈活管理錄音流程，避免錄到不需要的片段**。

**詳細說明：**

- 主要控制按鈕：
  - **開始錄音**（⏺）：點選後開始錄音，按鈕變為暫停圖示
  - **暫停 / 繼續**（⏸ / ▶）：切換錄音暫停與繼續
  - **停止錄音**（⏹）：結束本次錄音，進入儲存流程
- 停止錄音後，引導使用者：
  - 輸入錄音標題（預設為日期時間）
  - 選擇錄音分類（可自訂）
  - 確認儲存或捨棄
- 錄音檔案格式：WebM 或 WAV（依瀏覽器支援）

---

### F5：標記管理與回顧

**使用者故事：**
> 作為**錄音使用者**，我希望**在錄音完成後能以列表方式查看所有標記，並點選標記跳到對應時間點回聽**，以便**快速整理錄音內容、摘取重點片段**。

**詳細說明：**

- 錄音完成後，提供標記清單頁面，包含：
  - 標記時間戳（可點選跳轉播放）
  - 標記種類（圖示 + 文字）
  - 備註內容
- 支援依標記種類篩選
- 支援編輯、刪除標記
- 支援匯出標記清單（CSV / JSON 格式）

---

### F6：錄音檔案管理

**使用者故事：**
> 作為**錄音使用者**，我希望**能瀏覽、搜尋、播放和刪除所有歷史錄音檔案**，以便**有效管理日積月累的錄音資料**。

**詳細說明：**

- 錄音列表頁面，顯示：
  - 錄音標題、日期時間、時長
  - 標記數量統計
  - 分類標籤
- 支援搜尋（依標題、分類）
- 支援播放錄音（內建音訊播放器）
- 支援刪除錄音（需二次確認）
- 支援下載錄音檔案

---

### F7：系統整合介面

**使用者故事：**
> 作為**系統管理者或進階使用者**，我希望**本錄音系統提供標準化的資料介面，能與語音轉寫、AI 摘要、訪談歷史管理等外部系統串接**，以便**建構完整的訪談資料處理工作流**。

**詳細說明：**

本系統需設計為模組化架構，預留與以下系統的整合介面：

| 整合目標系統 | 整合方式 | 說明 |
|------------|---------|------|
| 語音轉寫與 API 整合 | RESTful API / 資料匯出 | 錄音完成後可觸發語音轉文字流程，並將標記時間戳對應到轉寫文本 |
| AI 摘要與報告產出 | 資料輸出介面 | 將錄音後設資料（標記、時間戳、備註）以結構化格式提供給 AI 摘要系統 |
| 訪談歷史與檢索管理 | 資料庫共用 / API | 錄音紀錄可被訪談管理系統索引、搜尋與歸檔 |

- 所有錄音與標記資料應以結構化格式儲存，便於匯出
- 提供 RESTful API 端點供外部系統存取錄音後設資料
- API 設計遵循一致的資料格式規範（JSON）

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Flask (Python) |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端音訊 | Web Audio API + MediaRecorder API |
| 前端波形 | Canvas API 或 Web Audio API AnalyserNode |
| 瀏覽器支援 | Chrome / Edge（桌面版，需支援 Web Audio API） |

### 3.2 效能考量

- **波形渲染延遲**：音量波形視覺化延遲應低於 100ms，確保即時回饋
- **標記回應時間**：點選標記按鈕後，標記應在 200ms 內被記錄並顯示回饋
- **錄音檔案大小**：單次錄音最長支援 4 小時（需考量瀏覽器記憶體限制）
- **頁面載入**：首頁載入時間不超過 3 秒
- **並發使用**：MVP 階段僅需支援單一使用者同時操作

### 3.3 安全考量

- 錄音檔案儲存於伺服器端，需設定適當的存取權限
- 使用者操作應有基本的身分驗證（帳號密碼登入）
- 防止未授權存取其他使用者的錄音資料
- 標記資料與備註不得包含 XSS 攻擊向量（需做輸入消毒）

### 3.4 使用性考量

- 介面應簡潔直覺，核心操作（錄音 + 標記）不超過 2 次點擊
- 波形 + 計時器 + 標記按鈕應在同一畫面內完整呈現，避免捲動
- 標記按鈕應足夠大，適合快速點擊（最小觸控目標 48x48px）
- 提供操作提示與鍵盤快捷鍵（例如空白鍵 = 暫停 / 繼續，數字鍵 = 快速標記）

## 4. MVP 範圍

### Must Have（最小可行產品）

> 這些功能是 MVP 必須完成的核心功能，缺少任何一項系統就無法正常使用。

- [ ] **錄音控制**：開始 / 暫停 / 繼續 / 停止錄音
- [ ] **即時音量波形**：畫面中央顯示即時波形，確認錄音進行中
- [ ] **錄音計時器**：頂部顯示 `HH:MM:SS` 計時
- [ ] **即時標記**：提供至少 3 種預設標記種類（關鍵重點、故事、不清晰），可在錄音中即時點選
- [ ] **標記時間戳記錄**：每個標記自動記錄對應的時間戳
- [ ] **錄音儲存**：錄音完成後可儲存檔案與標記資料

### Should Have（重要但非必要）

> 這些功能會顯著提升使用體驗，建議在 MVP 完成後優先開發。

- [ ] **自訂標記種類**：使用者可新增 / 編輯 / 刪除標記種類
- [ ] **標記備註**：標記時可附加簡短文字備註
- [ ] **標記回顧與跳轉**：錄音完成後以列表檢視標記，點選跳轉播放
- [ ] **錄音列表管理**：瀏覽、搜尋、刪除歷史錄音
- [ ] **波形標記視覺化**：在波形圖上顯示標記位置旗標
- [ ] **鍵盤快捷鍵**：空白鍵暫停、數字鍵快速標記

### Nice to Have（加分項目）

> 這些功能屬於進階需求，可在後續版本中實現。

- [ ] **系統整合 API**：提供 RESTful API 供語音轉寫 / AI 摘要 / 訪談管理系統存取
- [ ] **標記匯出**：匯出標記清單為 CSV / JSON
- [ ] **錄音分類管理**：為錄音設定自訂分類標籤
- [ ] **使用者帳號系統**：註冊、登入、多人隔離
- [ ] **錄音檔案下載**：下載錄音檔案至本地
- [ ] **深色模式**：支援淺色 / 深色主題切換

## 附錄：使用者操作流程摘要

```
使用者打開系統
    ↓
進入錄音主頁面（波形 + 計時器 + 標記按鈕）
    ↓
點選「開始錄音」
    ↓
┌──────────────────────────────────────┐
│  錄音進行中                           │
│  - 波形即時跳動                       │
│  - 計時器持續計時                      │
│  - 使用者隨時點選標記按鈕標記重點        │
│    └→ 自動記錄時間戳 + 標記種類         │
│    └→ 可選擇性輸入備註                 │
│  - 可暫停 / 繼續                      │
└──────────────────────────────────────┘
    ↓
點選「停止錄音」
    ↓
輸入標題 → 儲存錄音 + 標記資料
    ↓
進入錄音回顧頁面
    ↓
查看標記清單 → 點選標記 → 跳轉回聽
```
